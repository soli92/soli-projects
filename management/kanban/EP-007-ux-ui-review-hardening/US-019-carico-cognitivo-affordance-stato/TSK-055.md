# TSK-055 — Progressive disclosure su dettaglio progetto (F#2)

## Descrizione

In `app/projects/[slug]/page.tsx` l'Overview renderizza inline l'intero AI_LOG/README (testo + code), causando information overload. Introdurre progressive disclosure: accordion/collapse per sezione, oppure troncamento con azione "mostra tutto". L'utente deve poter scegliere cosa espandere; il default non deve dumpare tutto il contenuto inline.

**Finding:** F#2 (MED) della review UX/UI.
**rubric_ref:** Nielsen-8 (estetica e design minimalista) · dimensione UI «gerarchia» · dimensione flusso «passi vs valore».

## Riferimenti

- US: [[US-019]]
- Epic: [[EP-007]]
- Wiki: [[ux-ui-review-protocol]]

## Criteri di done

- [ ] L'Overview non renderizza più l'intero AI_LOG/README inline in un unico blocco.
- [ ] Ogni sezione è collassabile (accordion/collapse) oppure troncata con affordance "mostra tutto".
- [ ] Lo stato espanso/collassato è chiaro e accessibile (aria-expanded, focus management).
- [ ] Nessuna regressione di rendering di testo e code block all'espansione.
