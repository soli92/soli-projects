# TSK-057 — Differenziare empty-state vuoto vs nessun risultato filtri (F#8)

## Descrizione

L'empty-state dei task mostra "Nessun task trovato con i filtri selezionati" anche quando la lista è semplicemente vuota (0 todo, nessun filtro attivo), comunicando un messaggio fuorviante. Differenziare il copy: caso "vuoto" (nessun task esistente) vs caso "nessun risultato per i filtri" (filtri attivi che non matchano nulla), selezionando il messaggio in base alla presenza di filtri attivi.

**Finding:** F#8 (LOW) della review UX/UI.
**rubric_ref:** Nielsen-2 (corrispondenza sistema/mondo reale) · Nielsen-9 (aiutare a riconoscere/recuperare dagli errori).

## Criteri di done

- [ ] Con 0 task e nessun filtro attivo viene mostrato un copy "vuoto" (es. "Nessun task ancora").
- [ ] Con filtri attivi e 0 risultati viene mostrato il copy "nessun risultato per i filtri".
- [ ] La scelta del copy è guidata dalla presenza di filtri attivi, non dal solo conteggio risultati.
