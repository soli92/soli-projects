# TSK-056 — Stato pending/disabled sul submit del login (F#7)

## Descrizione

Il bottone "Entra" del login non ha loading/disabled state durante il submit della server action: l'utente può cliccare ripetutamente e non ha feedback che la richiesta è in corso. Aggiungere stato pending (es. `useFormStatus`) che disabilita il bottone e mostra un indicatore di caricamento durante l'esecuzione della server action.

**Finding:** F#7 (LOW-MED) della review UX/UI.
**rubric_ref:** Nielsen-1 (visibilità dello stato del sistema) · affordance «loading».

## Riferimenti

- US: [[US-019]]
- Epic: [[EP-007]]
- Wiki: [[ux-ui-review-protocol]]

## Criteri di done

- [ ] Durante il submit il bottone "Entra" è disabilitato.
- [ ] Durante il submit è mostrato un indicatore di stato (spinner/testo "Accesso in corso…").
- [ ] Lo stato pending è derivato dallo stato reale della server action (es. `useFormStatus`), non da un timer.
- [ ] Doppio submit accidentale impedito.
