# EP-004 — Infrastruttura & CI/CD Cross-progetto

## Obiettivo

Standardizzare i workflow CI/CD attraverso i repository soli92, implementare health check uniformi, e documentare runbook operativi per ogni provider di deploy.

## Contesto

L'ecosistema usa 3 provider: Vercel (frontend Next.js), Render (backend Express), Oracle ARM (soli-platform Docker) [^src: raw/soli-platform-agents.md §Repo]. I workflow GitHub Actions variano per repo: alcuni hanno lint+typecheck+test+build, altri solo deploy [^src: raw/soli-projects-agents.md §CI / GitHub Actions]. Non esiste monitoring unificato.

## Storie

- [[US-010]] — Standardizzazione workflow CI cross-progetto
- [[US-011]] — Health check e monitoring unificati
- [[US-012]] — Runbook deploy per ogni provider

## Rischi

- Disallineamento tra step CI locali e CI remota (es. manca npm test in deploy.yml).
- Health check assenti su backend Render (latenza cold start non monitorata).
- Oracle free tier: provisioning fragile e non automatizzato.
