# US-011 — Health check e monitoring unificati

## Descrizione

Come **developer**, voglio che ogni backend esponga un endpoint `/health` standardizzato e che ci sia un sistema di monitoring minimo per verificare uptime e latenza delle app in produzione.

## Criteri di accettazione

- [ ] Ogni backend (soli-agent, soli-prof, casa-mia-be, soli-dm-be, soli-platform services) ha `/health`.
- [ ] Smoke test suite eseguibile per verificare deploy production.
- [ ] Documentazione monitoring (strumenti, frequenza, responsabili).

## Task

- [[TSK-031]] — Standardizzare endpoint /health per ogni backend
- [[TSK-032]] — Creare smoke test suite production
- [[TSK-033]] — Documentare uptime monitoring
