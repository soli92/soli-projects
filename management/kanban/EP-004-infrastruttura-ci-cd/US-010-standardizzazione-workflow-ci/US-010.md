# US-010 — Standardizzazione workflow CI cross-progetto

## Descrizione

Come **developer**, voglio che ogni repository abbia un workflow CI consistente (lint, typecheck, test, build) allineato a un template condiviso, riducendo il rischio di deploy con test rotti.

## Criteri di accettazione

- [ ] Template workflow CI documentato (step minimi: lint, typecheck, test, build).
- [ ] Audit completato: quali repo soli92 hanno step mancanti.
- [ ] Step `npm test` aggiunto a workflow che lo omettono.

## Note tecniche

Il repo di collaborazione [[agentic-value-investor-application]] adotta pattern CI avanzati (Testcontainers per test di integrazione con DB reale, OpenAPI contract check automatico, Docker smoke build, dual Gradle+npm pipeline) documentati in [[deployment-patterns]]. Questi pattern sono un **riferimento di lettura** per migliorare la CI dei repo soli92, ma il template condiviso e l'audit si applicano solo ai repo soli92-owned. soli-projects non push CI template verso repo esterni.
